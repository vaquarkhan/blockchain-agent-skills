# Support

Use the following paths when you need help with the repository:

- bug reports: open a GitHub issue with reproduction details
- feature requests: open a GitHub issue explaining the use case and expected outcome
- adoption questions: start with `docs/getting-started.md` and the setup guide for your tool
- contribution questions: review `CONTRIBUTING.md`, `docs/skill-anatomy.md`, and `docs/preset-anatomy.md`

## Before Opening an Issue

1. verify you are using the latest files from `main` or the latest release
2. check the relevant setup guide in `docs/`
3. check `skills-index.md` to confirm the right skill bundle was chosen
4. include logs or command output when reporting installer, hook, or packaging failures
